Simple Banking System | Simple Banking System

Project Name:

  • Simple Banking System
    • Project Manager and Leader:
  • Sharon Mungania
    • Problem Project will Address:
  • This system aims at solving the problem of unresponsive and complex interfaces often users experience in using their current banking systems. My objective is to create a system that reduces the frequency with which consumers must get in touch with customer service while doing daily routines. Users will be able to safely login in and out of the system, access their transaction history, check their balance, and transfer money amongst registered platform users. Join me in implementing and providing this solution to realize customer satisfaction.
    • Project Goals:
  • To create a user-friendly banking system that offers essential banking functions for users.
  • To ensure security in the access and transfer of banking accounts and money.
  • To ensure users can transfer funds among registered platform users seamlessly.
    • Project Objectives and Scope: Objectives:
  • Allow only authorized users to safely sign in and out of the system.
  • Allow users to view their bank balance and transaction history real-time.
  • Allow users to deposit funds seamlessly.
  • Update bank balance real-time as the transactions take place.
    • Scope:
  • This platform will implement the following technologies; for frontend; HTML5, Javascript(React) and CSS3, and for backend; Nodejs, Javascript, and MySQL for database management, and Railway for cloud and hosting.
  • The system features will include; authorized user sign in and sign out, balance checks, deposits, fund transfers and data storage.
    • Key Deliverables:
  • A fully functional web system for the Simple Banking System.
  • User authentication module (sign-in and sign-out).
  • Fund deposit functionality.
  • Balance checks feature.
    • Proposed Schedule:
  • Week 1: Requirements gathering.
  • Week 2: Designing system architecture, wireframe, and prototypes.
  • Week 3: Developing user authentication feature.
  • Week 4: Developing balance check and deposit functionality.
  • Week 5: Develop fund transfer functionality.
  • Week 6: Testing and debugging all features thus far.
  • Week 7: Prepare documentation for user education and training.
  • Week 8: Project review and presentation.
    • Potential Limitations and Setbacks:
  • Potential system development time delay due to any unforeseen challenges.
  • Scope creep from user feedback.
  • Security issues arising during testing and implementation.
